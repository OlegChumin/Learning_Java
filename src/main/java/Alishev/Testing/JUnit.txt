1. test case - помечается аннотацией @Test
2. Вспомогательные аннотации @Before @BeforeClass, @After, @AfterClass
3. Тестовый класс называют с суффиксом Test (например Vector2DTest)
4. тестовые методы должны содержать слово should в названии (sumShouldBePositive)
5. все тестовые методы public void (не static)
6. дополнительные параметры аннотации @Test (expected и timeout)
7. класс Assert -> assertTrue(), assertFalse(), assertEquals(), assertArrayEquals(), assertNotEquals(), assertSame(),
assertNotSame(). (все методы статические).
8. Аннотации @Ignore (тест не выполняется)
9. Вместо Assert можно использовать Assume(assumeFalse(), assumeEquals()...) - если проверка верна то тест пройден,
если проверка неверна то тест игнорируется а в Assert тест валится!
10. @BeforeClass - запускается только один раз при запуске теста
@Before - запускается перед каждым тестовым методом (для данных - вспомогательные данные)
@After - запускается после каждого метода
@AfterClass - запускается после того, как отработали все тестовые методы

Жизненный цикл тестирующего класса
- @BeforeClass (статический)- метод с этой аннотацией запускается 1 раз для тестового класса (не каждый раз для метода)
- создается объект (экземпляр) тестового класса_Test
- @Before - запускается перед каждым тестовым методом
- @Test - запуск тестового метода
- @After - запускается для каждого тестового метода
- @AfterClass - запускается для тестового класса после выполнения всех тестовых методов
